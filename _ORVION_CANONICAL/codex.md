# codex.md — Superseded

Status: Deprecated. Do not use this file as an authority for agent conduct.

## Why

This file previously operated as a second, independent statement of agent
behavior (`Priority: Highest`, `Status: Canonical`), built before `AGENTS.md`
and `PROTOCOL.md` existed, and was never reconciled against them once they
did. Its process and conduct sections — Never Assume, Scope Control, Working
Memory, Think Before Coding, Coding Rules, Architecture Rules, Error
Philosophy, Long-Term Rule — duplicated `AGENTS.md`/`PROTOCOL.md`/
`global-rules.md`/`CODING_STANDARDS.md` almost exactly, in different words,
with no stated precedence between the two documents. Two documents both
claiming unqualified top-level authority over agent conduct is exactly the
kind of ambiguity a new participant cannot resolve without being told which
one wins.

## What to use instead

`AGENTS.md` is the operational authority for agent execution. `PROTOCOL.md`
states collaboration principles and defers to `AGENTS.md` explicitly.
`global-rules.md` supplements `AGENTS.md` and does not override it.

This file's business-philosophy content — the concrete service examples,
the travel-agency role perspective, the event and UI/API philosophy — was
not duplicated anywhere else and has been preserved, not discarded: see
`PROJECT_CONTEXT.md` §12, "Business Context (from codex.md)".

## Do not

- Do not treat this file as authoritative for agent conduct.
- Do not restore its process sections; they are covered by `AGENTS.md`/`PROTOCOL.md`.
